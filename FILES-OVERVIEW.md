## 🎨 Decap CMS (Админ-панель)

- **Доступ:** https://ваш-сайт.ru/admin

### `/public/admin/config.yml`
- **Что:** Конфигурация Decap CMS
- **Содержит:**

- **Документация:** https://decapcms.org/docs/configuration-options/

## ⚙️ Автоматизация

### `.github/workflows/deploy-timeweb.yml`
- **Что:** GitHub Actions workflow для деплоя на Timeweb
- **Когда запускается:** При push в master или вручную
- **Что делает:**
  1. Устанавливает зависимости (`npm ci`)
  2. Собирает сайт (`npm run build`)
  3. Загружает на Timeweb через FTP
- **Требует GitHub Secrets:** FTP_SERVER, FTP_USERNAME, FTP_PASSWORD

### `.github/workflows/deploy-netlify.yml.example`
- **Что:** Альтернативный workflow для Netlify
- **Использование:** Переименовать в `.yml` если используете Netlify
- **Преимущество:** Проще, не требует FTP

### `.github/workflows/README.md`
- **Что:** Документация по GitHub Actions workflows
- **Для кого:** Разработчик

### `scripts/merge-cms-json.js`
- **Что:** Node.js скрипт для объединения JSON файлов
- **Зачем:** Decap CMS создаёт отдельные файлы, сайт ожидает массивы
- **Когда запускается:** Автоматически перед сборкой (prebuild)
- **Что делает:**
  - Находит файлы `news-*.json`, `project-*.json` и т.д.
  - Объединяет их в `news.json`, `projects.json`
  - Сортирует по ID
