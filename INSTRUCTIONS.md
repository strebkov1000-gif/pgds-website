## Как редактировать контент:

Все данные хранятся в папке **data/** в формате JSON:

### 1. Информация о компании
Файл: `data/content.json`
- Основная информация о компании
- Услуги
- Партнеры
- Регионы присутствия
- Оборудование

### 2. Проекты
Файл: `data/projects.json`
Добавьте новый проект:
```json
{
  "id": "4",
  "title": "Название проекта",
  "description": "Описание проекта",
  "category": "Тип объекта",
  "status": "Завершен или В процессе",
  "year": 2025,
  "location": "Город",
  "image": "/images/projects/project-4.jpg"
}
```

### 3. Новости
Файл: `data/news.json`

### 4. Вакансии
Файл: `data/vacancies.json`

### 5. Тендеры
Файл: `data/tenders.json`

## Как работать с сайтом:

### Запуск сервера разработки:
```bash
cd pgds-website
npm run dev
```
Откройте http://localhost:3000

### Остановка сервера:
Нажмите `Ctrl + C` в терминале

### Редактирование контента:
1. Откройте нужный JSON файл в папке `data/`
2. Внесите изменения
3. Сохраните файл
4. Обновите страницу в браузере (изменения применятся автоматически)

### Добавление изображений:
1. Поместите изображения в папку `public/images/`
2. Используйте путь `/images/название-файла.jpg` в JSON

## Следующие шаги:

### 1. Добавьте реальные данные:
- Замените контактные данные в `data/content.json`
- Добавьте реальные проекты в `data/projects.json`
- Загрузите фотографии проектов, техники, логотипы партнеров

### 2. Настройте изображения:
- Создайте папки в `public/images/`:
  - `projects/` - фотографии проектов
  - `partners/` - логотипы партнеров
  - `news/` - изображения новостей
  - `equipment/` - фотографии техники

### 3. Подготовка к публикации:
Когда сайт будет готов к публикации:
```bash
npm run build
npm start
```

### 4. Развертывание на хостинге:
Рекомендуем использовать Vercel (бесплатно):
1. Зарегистрируйтесь на https://vercel.com
2. Подключите проект
3. Нажмите "Deploy"

## Доступ к админ-панели:

URL: http://localhost:3000/admin

Админ-панель предоставляет:
- Обзор всего контента
- Быстрый доступ к файлам данных

## Важные файлы:

- `data/` - все данные сайта

- `components/` - переиспользуемые компоненты
